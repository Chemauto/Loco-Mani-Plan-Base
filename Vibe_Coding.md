# Vibe Coding 规范

你是一个 AI 编程工具，你在修改本项目时应遵循以下规范：

## 1. 最小改动原则

- 完成任务的基础上保证改动最少，不做任何与任务无关的修改
- 不擅自添加功能、注释、错误处理或重构已有代码
- 不删除或修改已有的空白行、格式，除非与任务直接相关

## 2. 风格统一

- 代码风格、命名习惯与仓库现有代码保持一致
- 文件编排方式（目录结构、命名规范）与现有模块对齐
- 新增文件放入对应模块目录，不随意创建新目录

## 3. 参数配置

- 所有可调参数统一使用 YAML 文件管理（如 `config.yaml`），不使用命令行参数
- 新增参数时放入对应模块的配置文件中，保持配置结构清晰

## 4. 语言规范

- 代码、注释使用英文
- 文档（README、Markdown）使用中文

## 5. 依赖管理

- 不擅自引入新的第三方依赖，确有需要时先提出并说明理由
- 优先使用仓库中已有的依赖和工具

## 6. 修改纪律

- 修改代码前先理解上下文，不盲目修改
- 配置文件（yaml/json）和仿真环境参数修改需格外谨慎
- 每次修改应说明改了什么、为什么改
